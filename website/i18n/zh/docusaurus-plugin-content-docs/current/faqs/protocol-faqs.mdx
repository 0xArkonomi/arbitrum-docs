# 常见问题解答：协议

### Arbitrum Rollup 和 Arbitrum AnyTrust 有什么区别？ {#q-rollup-vs-anytrust}

Arbitrum Rollup 是一种 Optimistic Rollup 协议； 它是无需信任和无需许可的。 这些属性的部分实现方式是要求所有链数据都发布在第 1 层。这意味着此数据的可用性直接取决于以太坊本身的安全属性，反过来，任何一方都可以参与验证链并确保其安全。

相比之下，Arbitrum AnyTrust 引入了信任假设以换取较低的费用； 数据可用性由数据可用性委员会 (DAC) 管理，这是一组固定的、经过许可的实体。 我们引入了阈值 K，假设至少有 K 个委员会成员是诚实的。 为简单起见，我们在此假设委员会的规模为 20，K 值为 2：

如果 20 名委员会成员中有 19 名*和* Sequencer 是恶意的并串通一气，他们就可以破坏链条的安全（例如，窃取用户的资金）； 这是新的信任假设。

如果只有 2 到 18 名委员会成员表现良好，AnyTrust 链将以“Rollup 模式”运行； 即，数据发布在 L1 上。

然而，在普遍的情况下，20 名委员会成员中至少有 19 名表现良好，系统运行时不会在 L1 上发布 L2 链的数据，因此用户支付的费用要低得多。 这是 AnyTrust 链相对于 rollups 的核心优势。

如果您正在等待最小化新的信任假设 AnyTrust 模型的变体； 敬请关注。

有关更多信息，请参阅 [Inside AnyTrust](/inside-anytrust.mdx)。

### “Sequencer”和“Validators”是一回事吗？ 中心化的 Sequencer 可以做坏事，比如偷走我所有的钱吗？ {#q-seq-vs-val}

两者皆是否定的

Arbitrum 链的排序器和验证器以及其他完全不同的实体，都具有自己不同的角色。

[Sequencer](/sequencer.mdx) 是被授予特定交易排序特权的实体； 一旦 Sequencer 提交了一个batch（通过在以太坊上发布一个批次），它就没有接下来会发生什么的决定权了（即执行）。 一个恶意的 Sequencer 可以做一些事情，比如重新排序交易或*暂时*延迟交易的包含（当然这些事情肯定是烦人和糟糕的），但不会做任何危及链安全的事情。

*验证者*负责链的安全； 即，对链状态提出质押声明，相互争议等。

目前，在 Arbitrum One 上，Sequencer 是一个由 Offchain Labs 维护的中心化实体。 最终，单一的 Sequencer 将被分布式的 Sequencer 委员会所取代，他们就交易排序达成共识。 此升级将是一个改进； 我们不希望您必须相信我们不会重新排序您的交易。 但是，严格来说，Arbitrum One 并不是 “必须要” 达到我们上述的这个最终状态的 。

换句话说：**_具有中心化排序器的 Arbitrum Rollup 链在理论上仍然可以是无信任的！_**

也就是说——比去中心化 Sequencer 更重要的事情，即你应该更关心的事情——去中心化 *Validators*。

Arbitrum One 的 Validator 集合目前已列入白名单； 随着时间的推移，白名单将增长，然后被完全删除。 有关详细信息，请参阅[“Mainnet Beta”](/mainnet-beta.mdx)。

### 如果有争议，我的 L2 交易是否会被重组/丢弃？ {#q-dispute-reorg}

不会; 一旦 Arbitrum 交易被包含在 L1 上，就无法对其进行重组（当然，除非 L1 本身重组）。 “争议”涉及验证者不同意执行，即链的输出状态。 但是，输入是无可争议的； 它们由 L1 上的收件箱确定。 （参见[交易生命周期](/tx-lifecycle.mdx)）

### ...好吧，但如果有争议，我的交易会延迟吗？ {#q-dispute-delay}

争议唯一会增加延迟的是 L2 到 L1 消息的确认。 所有其他交易将继续处理，即使争议仍在进行中。 （另外：在实践中，大多数 L2-to-L1 消息代表同质化代币资产的提取；这些可以通过第三方“流动性退出”桥以不信任的方式完成*即使在争议期间*退出。参见[L2-to-L1 消息]（/ arbos/l2-to-l1-messaging.mdx)）。

### 为什么选择“一周”作为 Arbitrum One 的争议窗口？ {#q-why-one-week}

通常，Arbitrum Rollup 需要一些争议期，以便验证者有时间对无效断言提出异议。

假设验证者在将他们的交易包含在 L1 中没有遇到困难，预计一周的时间足以让验证者进行互动争议。 根据以太坊研究社区——以及其他第 2 层项目——的普遍共识，选择了一周，以便在协调的以太坊利益相关者审查攻击的情况下为社区提供足够的时间进行社会协调。
